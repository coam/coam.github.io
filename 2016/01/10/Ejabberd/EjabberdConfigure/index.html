Ejabberd 配置相关

2. ejabberd 配置文件

ejabberd主要两个配置文件 /etc/ejabberd/ejabberdctl.cfg 和 /etc/ejabberd/ejabberd.yml

其中 配置文件 /etc/ejabberd/ejabberdctl.cfg 中有一配置项 

#ERLANG_NODE=ejabberd@localhost
#需改当前服务器的域名为
ERLANG_NODE=ejabberd@coopens.com

修改了 ejabberdctl.cfg 中的 域名后,需要同时在 ejabberd.yml 中增加虚拟主机 coopens.com

hosts:
  - "localhost"
  - "coopens.com"

给 ejabberd 添加管理员

使用命令 sudo ejabberdctl register 添加用户 zyf 后,修改 ejabberd.yml 以下两段:

acl:
  admin:
    user:
      - "zyf": "coopens.com"
access:
  configure:
    admin: allow

访问 http://coopens.com:5280/admin 管理 ejabberd 服务器

修改 ERLANG_NODE 后使用命令 ejabberdctl live 启动 ejabberd 会报如下错误: 

[critical] Node name mismatch: I'm [ejabberd@coopens.com], the database is owned by [ejabberd@localhost]

原因是 Mnesia 和 新配置的 node 名称不匹配,如果刚安装没有重要数据的话,可以将 /var/lib/ejabberd/* 下的所有 spool 文件全部删除,使用重启即可重新生成新的数据文件

Mysql config problem
How to change bydefault node name of ejabberd which is ejabberd@localhost?

使用命令注册用户

sudo ejabberdctl register zyf coopens.com 123456

出现如下错误:

Can't set long node name!\nPlease check your configuration

原因可能是 /etc/hosts 配置有问题

#127.0.0.1 localhost coam
127.0.1.1   localhost.localdomain   localhost coam coopens.com

改为

#127.0.0.1 localhost coam
127.0.1.1   localhost.localdomain localhost coam coopens.com

重新执行以下命令重新注册用户就OK了

sudo ejabberdctl register zyf coopens.com 123456

二. 需要引入其它域名,只需在配置文件 /etc/ejabberd/ejabberd.yml 里的配置段

hosts:
  - "localhost"
  - "coam.co"
  - "coopens.com"

添加其它域名即可使用以下命令添加用户,并可跨域聊天加好友

ejabberdctl register zyfmix coam.co ******
